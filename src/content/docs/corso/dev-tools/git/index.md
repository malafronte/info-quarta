Git è un sistema di controllo di versione distribuito (Distributed Version Control System - DVCS) creato da Linus Torvalds nel 2005. È diventato lo standard de facto per la gestione del codice sorgente nel mondo dello sviluppo software moderno. La sua importanza risiede nella capacità di tracciare ogni singola modifica apportata ai file di un progetto, permettendo agli sviluppatori di "viaggiare nel tempo" per recuperare versioni precedenti, confrontare cambiamenti e capire chi ha modificato cosa e perché.

A differenza dei sistemi centralizzati, Git permette a ogni sviluppatore di avere una copia completa della storia del progetto sul proprio computer locale. Questo approccio garantisce una maggiore velocità nelle operazioni quotidiane e la possibilità di lavorare offline senza dipendere da un server centrale. Inoltre, le funzionalità di branching e merging di Git sono estremamente potenti e flessibili, facilitando il lavoro parallelo su nuove funzionalità (feature branches) senza interferire con il codice stabile in produzione.

In questa sezione del corso, non ci limiteremo a esplorare i comandi fondamentali di Git, ma approfondiremo anche l'utilizzo di GitHub. GitHub è una piattaforma di hosting per progetti software che utilizza Git per il controllo di versione, aggiungendo funzionalità social e di collaborazione come il tracciamento dei bug (issue tracking), le richieste di modifica (pull requests) e l'automazione dei flussi di lavoro (GitHub Actions). Impareremo come creare repository remoti, sincronizzare il lavoro locale con il cloud e collaborare efficacemente con altri membri del team.
